# J.A.R.V.I.S. · Air Draw

Рисуешь в воздухе указательным пальцем перед веб-камерой. Браузер, никаких установок, никакого железа.

![JARVIS HUD — тёмно-синий фон, неоновый циан](.github/preview.png)

---

## Быстрый старт

```bash
# нужен любой статический сервер — getUserMedia не работает через file://
cd neon-air-draw
python3 -m http.server 8000
# открыть http://localhost:8000/
```

Или через Node:

```bash
npx serve neon-air-draw
```

Разреши доступ к камере. Когда появится изображение и задёргается счётчик FPS — всё работает.

**Или без сервера** — просто двойной клик на `neon-air-draw-standalone.html`. Это тот же проект, собранный в один файл скриптом `build_standalone.py`.

**На телефоне** — открывай `neon-air-draw-standalone.html` напрямую или подними сервер и зайди по локальной сети. Мобильные браузеры требуют `https://` для камеры; исключение — `localhost` с USB-отладкой.

## Жесты

| Жест | Что делает |
|------|-----------|
| ☝ указательный палец | рисует текущим цветом |
| 🖐 открытая ладонь (внутренней стороной к камере) | ластик (радиус — слайдер внизу) |
| 🖐 удержать ладонь 1 секунду | полная очистка холста |
| 🤝 три пальца щепотью (большой + указательный + средний) | захватить связную компоненту под пальцами |
| 🤝 захват + открытая ладонь второй руки | вращать захваченный фрагмент |
| 🤝🤝 щепоть двумя руками | масштаб + панорамирование всего рисунка |
| 🗑 все пять пальцев щепотью → резкий бросок | скомкать рисунок в бумажный шарик с физикой |
| 🤘 коза (указательный + мизинец) | следующий цвет в палитре |
| `Ctrl / Cmd + H` | стрим-режим — прячет весь HUD для захвата в OBS |
| ⟳ кнопка (мобильные) | переключить фронтальную / заднюю камеру |

### Распознавание

После паузы ~0.7 с приложение пытается разобрать что нарисовано:

- **один штрих** → геометрия: LINE / CIRCLE / RECT. Если похоже — грубой набросок заменяется чистой неоновой фигурой.
- **несколько штрихов** → Tesseract (рус + eng). При уверенности ≥ 55% штрихи заменяются распознанным текстом в том же цвете.
- **ничего не распознали за 3.5 с** → штрихи остаются как есть.

## Структура

```
neon-air-draw/
├── index.html                       # модульная версия (нужен сервер)
├── neon-air-draw-standalone.html    # всё в одном файле (работает через file://)
├── build_standalone.py              # собирает standalone из модулей
├── css/
│   └── styles.css                   # тема JARVIS HUD
└── js/
    ├── main.js                      # DOM, состояние, основной цикл onResults
    ├── gestures.js                  # геометрия ладони и классификаторы поз
    ├── recognition.js               # классификатор фигур (линия / круг / прямоугольник)
    ├── stroke-draw.js               # гладкий штрих через Catmull-Rom
    ├── paper-ball.js                # процедурный бумажный шарик с кешем
    ├── one-euro.js                  # 1€-фильтр для ландмарков (Casiez et al., 2012)
    └── hand-tracker.js              # MediaPipe Tasks Vision + getUserMedia
```

Два варианта одного и того же кода. В `index.html` модули загружаются отдельно — удобно для разработки, требует сервера. В standalone всё инлайнится скриптом `build_standalone.py`, браузерные CORS-ограничения на `file://` не мешают.

## Мобильные устройства

Работает на Android Chrome и iOS Safari 16.4+. Несколько отличий от десктопа:

- **Разрешение камеры** снижается до 640×360 (вместо 960×540) — на мобильном CPU это снижает нагрузку примерно вдвое без заметной потери в качестве детекции.
- **Пороги уверенности** чуть ниже (0.50/0.45/0.35 вместо 0.55/0.50/0.40) — мобильные камеры хуже справляются с плохим освещением и дрожанием руки.
- **Кнопка ⟳** переключает фронтальную и заднюю камеры. Задняя удобна если телефон лежит на столе — можно рисовать над ним.
- **HUD адаптируется**: меньше отступы, кнопки увеличены до минимального тач-таргета 44px, подсказка жестов скрыта (не влезает).

Жест двойной щепоткой (зум) на телефоне работает, но неудобен — держать телефон и одновременно делать жест двумя руками сложно. Лучше заранее выбрать масштаб на десктопе или просто рисовать крупно.

## Как это работает внутри

Самое неочевидное решение — четыре canvas-слоя вместо одного:

```
artCanvas      — сохранённые штрихи и фигуры (постоянный)
pendingCanvas  — штрихи ожидающие распознавания
liveCanvas     — активные штрихи (перерисовывается каждый кадр целиком)
──────────────────────────────────────────────────────────────────────
viewCanvas     — видимый элемент; каждый кадр очищается и композитит всё
```

Зачем три промежуточных холста? Если рисовать штрих инкрементально — дорисовывать новый сегмент к уже нарисованному — `shadowBlur` накапливается на каждом стыке и линия выглядит как ожерелье из капель. `liveCanvas` решает это грубо: стирается и перерисовывается целиком каждый кадр, Catmull-Rom строит один `Path2D` по всем точкам. `pendingCanvas` нужен чтобы не смешивать «черновик» с `artCanvas` пока распознаватель ещё думает.

Штрих — три прохода по одному пути: полупрозрачный ореол (lineWidth 10), цветной слой (4px), белая сердцевина (1.5px). Никаких shadowBlur, никаких стыков.

Жесты классифицируются через проекцию кончиков пальцев на ось ладони (запястье → MCP среднего пальца). Благодаря этому кулак — кулак независимо от того, как повёрнута рука. Без этого трюка пришлось бы привязываться к абсолютным y-координатам, и жесты ломались бы при любом наклоне.

Ландмарки от MediaPipe дрожат — особенно когда рука почти неподвижна. 1€-фильтр (Casiez et al., 2012) с параметрами `mincutoff=4.5` и `beta=0.85` давит дрожь в покое и почти полностью отключается на быстрых движениях. Результат: линия не трясётся и не запаздывает.

При переключении камеры (кнопка ⟳) фильтры сбрасываются — иначе они пытаются интерполировать от старых координат к новым и дают секундный «дрифт».

## Зависимости (CDN, без установки)

| Библиотека | Зачем |
|-----------|-------|
| [`@mediapipe/tasks-vision`](https://developers.google.com/mediapipe) `0.10.14` | HandLandmarker, 21 ландмарк на руку, WebGL-делегат |

Модель рук грузится с `storage.googleapis.com` при первом запуске (~1.5 МБ). Нужен интернет.

## Сборка standalone

```bash
python3 build_standalone.py
```

Скрипт берёт все JS-модули из `js/`, убирает `export` / `import`-строки, склеивает в один `<script type="module">`, инлайнит CSS. Получается один HTML-файл без внешних зависимостей на файловую систему.

---

## Браузеры

Работает в Chrome/Edge 113+. Firefox — работает на CPU-делегате. Safari 17+ — работает, но `filter: blur()` в `destination-out` может вести себя иначе на некоторых версиях.

Страница должна открываться через `https://` или `localhost` — иначе `getUserMedia` заблокирован.

## Куда смотреть в коде

Если хочешь разобраться или что-то поменять — `js/main.js` функция `onResults`. Приоритеты жестов там выделены комментариями `===== ПРИОРИТЕТ N =====`, читается как блок-схема.

Логика самих жестов без единого обращения к DOM — `js/gestures.js`. Можно запустить в Node и написать тесты.

Штрих — `js/stroke-draw.js`, всего 44 строки. Фильтр — `js/one-euro.js`, 24 строки.
